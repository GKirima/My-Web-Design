Project 2: Building a Secure 3-Tier Architecture on AWS üåê

Overview:
This project showcases the deployment of a classic 3-tier architecture on AWS, designed for scalability, high availability, and security. The architecture separates the presentation, application logic, and data layers across different network tiers.

1. VPC and Subnet Setup:
- Create a VPC with CIDR block 10.0.0.0/16.
- Create 6 subnets across 2 Availability Zones:
‚Ä¢ 2 Public Subnets (Web Layer)
‚Ä¢ 2 Private Subnets (App Layer)
‚Ä¢ 2 Private Subnets (DB Layer)

2. Internet Gateway and NAT Gateway:
- Attach an Internet Gateway to allow public internet access for web tier.
- Set up a NAT Gateway in public subnet for private subnets to access the internet securely.

3. Route Tables:
- Create route tables and associate them appropriately with each subnet.
- Public subnets route 0.0.0.0/0 to Internet Gateway.
- Private subnets route 0.0.0.0/0 to NAT Gateway (for app layer).

4. Security Groups:
- Web SG: Allow HTTP/HTTPS from the internet, SSH from admin IP.
- App SG: Allow traffic only from Web SG.
- DB SG: Allow traffic only from App SG on port 3306 (MySQL).

5. Launch EC2 Instances:
- Web Layer: EC2s in public subnets (Apache/Nginx).
- App Layer: EC2s in private subnets (Node.js, Python, etc).
- DB Layer: Amazon RDS in DB subnets (Multi-AZ enabled).

6. Load Balancing and Auto Scaling:
- Use an Application Load Balancer for the Web Tier.
- Configure Auto Scaling Group for the web and app tiers.
- Health checks and target groups configured for each tier.

7. Database Configuration:
- Amazon RDS (MySQL/PostgreSQL) deployed in the DB tier.
- Enabled Multi-AZ for high availability and automatic failover.
- DB security group restricted to App layer only.

8. Monitoring and Logging:
- Enable CloudWatch for instance, application, and database monitoring.
- Enable VPC flow logs for traffic auditing.
- Configure alarms and dashboards for key metrics (CPU, latency, etc).

9. Testing:
- Test load balancer from browser.
- Verify application logic works between tiers.
- Confirm no direct access to the app or DB layer from the public internet.

10. Conclusion:
This architecture enables modular deployment, ease of scaling, and layered security. It's ideal for modern web applications requiring isolation and availability.

üí° Pro Tips:

  • Always place databases in private subnets.
  • Use IAM roles to grant permissions instead of hardcoding credentials.
  • Use CloudFormation or Terraform to automate infrastructure deployment.
  • Enable termination protection for critical instances and databases.
  • Regularly review security group rules and enable least privilege access.
