Step 1: Create Your Static Website

Start by building your site with an index.html file and optional styles.css and images/ folder.

/my-website
│
├── index.html
├── styles.css
└── images/

Step 2: Host on GitHub Pages

  1. Create a GitHub account and a new repository (e.g., my-website).
  2. Push your project files to the repo using Git:
    3. git init
git remote add origin https://github.com/yourusername/my-website.git
git add .
git commit -m "Initial commit"
git push -u origin main
  3. Go to Settings → Pages in your repository.
  4. Select main branch and click "Save."
  5. Your site is live at: https://yourusername.github.io/my-website

Step 3: Host on AWS S3 (Static Website Hosting)

  1. Login to AWS Console and go to S3 service.
  2. Create a new bucket (e.g., my-static-site).
  3. Uncheck "Block all public access" during bucket creation.
  4. Enable static website hosting in bucket Properties.
  5. Upload your files (index.html, etc.) to the bucket.
  6. Add the following Bucket Policy to allow public access:
    7. {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-static-site/*"
    }
  ]
}
  7. Access your website using the S3 endpoint URL provided.

Step 4: (Optional) Point a Custom Domain

  1. Register a domain via Route 53 or any registrar.
  2. Create an alias or CNAME record to your S3 endpoint (or GitHub Pages).
  3. Use Amazon Certificate Manager (ACM) for HTTPS (if using CloudFront).

You're Live!

Congratulations! Your static site is now live and accessible to the public.
